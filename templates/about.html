About the Drug Information Database

Welcome to the Drug Information Database. This web application helps users search for drugs, recommend medicines based on symptoms, and provide detailed information about various drugs.

Features

  • Search for Drugs: Use the search functionality to look up drugs by name or description.
  • Symptom-based Recommendations: Enter your symptoms to receive medicine recommendations from the system.
  • Add New Drugs: Administrators can add new drugs to the database using the 'Add Drug' functionality.
  • Medical Information: View in-depth information about drugs, their uses, and side effects.

How to Use

  1. Search for Drugs: Navigate to the search page, enter a drug name or a symptom, and view relevant results.
  2. Get Recommendations: Use the symptom input to receive suggestions on what medicines may help alleviate your symptoms.
  3. View Details: Each drug result provides detailed information, including its uses, side effects, and more.

  # app.py - Main Logic:
  - The app.py file sets up the core Flask application.
  - It configures the database URI to use MySQL for storing drug and symptom data.
  - Routes handle interactions: 
     /add_drug allows users to input new drug information.
     /search_drug enables users to search and retrieve information.
  - It integrates AJAX for smooth, real-time search functionality and symptom-matching logic.
  
  # templates:
  - home.html: The main page has a search bar where users input symptoms or drug names. It is linked to the drug recommendation system.
  - add_drug.html: Allows users to add information for new drugs, including name, symptoms treated, side effects, etc.
  - search.html: Displays drug details when a user searches for specific drugs or symptoms.
  
  # Database Structure:
  - The MySQL database contains tables such as:
    Drug, Symptom, Disease, and BodySystem. 
    These tables store all necessary information like drug names, symptoms treated, and diagnoses.
    
  - The Drug model holds drug data, which is cross-referenced with Symptom and Disease tables.
  
  # Logic:
  - When a user enters symptoms, the system searches the Symptom table to match the symptoms. 
  - If no diseases match, the system looks for drugs that treat the symptoms, pulling from the drug_database.
  - If the symptom matches a Disease, information like diagnosis and treatment options are displayed instead of drugs.
  
  # Frontend:
  - The UI is interactive, with modern features like hover effects, a progress bar, and buttons styled for quick access.
  - The chat feature is embedded at the bottom of the homepage, allowing users to input symptoms and get quick responses.
  
  # search_drug AJAX Call:
  - When a user searches for a drug, an AJAX request fetches drug details from the database without refreshing the page.
  - It enhances user experience, making the search dynamic and faster.
  
  # Conclusion:
  The Drug Information Database is designed to simplify the process of finding drug and symptom-related data with an interactive, user-friendly interface.
